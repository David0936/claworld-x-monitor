📖 使用教程

去设置页填写 →

四步即可跑起来：拿一个 X 数据源 Key → 设博主 → （可选）配飞书推送 → （可选）配 AI 翻译解读。所有配置都在 设置 页。

获取 X(Twitter) 数据源 API Key 必填

本工具不直连官方 Twitter API，而是用更便宜的第三方聚合源 twitterapi.io，按调用量计费、无需开发者审核。

  1. 打开 https://twitterapi.io，注册并登录。
  2. 在控制台（Dashboard）里找到 API Key，复制。
  3. 充值少量额度即可（按请求计费，监控几个博主很便宜）。
  4. 把 Key 粘到设置页的 TWITTER_API_KEY 里保存。
⚠️ 注意别和名字相近的 TwtAPI / twtapi.io 搞混，Key 不通用。
设置要监控的博主 必填

在设置页的 监控账号 TARGET_ACCOUNTS 里填 X 用户名（@ 后面那串，不是昵称）。

  • 多个博主用英文逗号分隔，例如：aleabitoreddit, DeItaone, unusual_whales
  • 填不填 @ 都行，系统会自动去掉。
  • 怎么找用户名：打开博主主页，网址 x.com/用户名 里的那段就是。
  • 抓取间隔 建议 60 秒以上；首次回溯小时数 决定第一次启动往前抓多久的推文。
默认已内置博主 @aleabitoreddit，可直接试跑。改完到「实时推文」页点停止→开始即可生效。
配置飞书推送 可选

命中的推文可自动推到飞书群（支持多群、按博主分流）。

  1. 飞书群 → 设置 → 群机器人 → 添加机器人 → 选「自定义机器人 Webhook」。
  2. 复制生成的 Webhook 地址；若开启了「签名校验」，把 签名密钥(Secret) 一并复制。
  3. 在设置页「飞书群」一栏新增一条：填备注、粘 Webhook、（如有）粘 Secret。
  4. 指定博主 留空＝该群收全部；填用户名＝只收这些博主的推文（分群路由）。
想全推可加一条不填博主的群；某条想手动补推，在推文详情页点「推送」。
或用 Telegram 推送 可选
  1. Telegram 里找 @BotFather → /newbot → 拿到 Bot Token。
  2. 把 bot 拉进群、或先给它发一条消息（否则它不能主动发你）。
  3. 浏览器打开 https://api.telegram.org/bot<你的token>/getUpdates，找 "chat":{"id": ...} 里的 chat_id（私聊=你的 id，群=负数，频道=@用户名）。
  4. 设置页「Telegram 推送」里新增一条，填 token + chat_id 即可，同样支持按博主分流。
配置 AI 翻译 / 中文标题 / 财经解读 可选

开启后，每条英文推文会自动生成中文标题、翻译和简要财经解读。支持两类服务，设置页一键切换：

A · 通义千问 / OpenAI 兼容接口 推荐
  • 提供商选 openai（兼容各类中转站 / 国产大模型）。
  • LLM_URL：接口地址，如通义 https://dashscope.aliyuncs.com/compatible-mode/v1。
  • LLM_API_KEY：对应平台的 Key。
  • LLM_MODEL：如 qwen-plus。
B · Claude 官方
  • 提供商选 claude。
  • CLAUDE_API_KEY：在 console.anthropic.com 创建。
  • CLAUDE_MODEL：如 claude-opus-4-8。
💡 用中转站时一律选 openai 提供商填 LLM 那一组，别选 Claude 单选钮。不配 AI 也能跑，只是推文不带中文标题/解读。
填完以上，去「实时推文」页点 开始监控 就跑起来了。 现在去设置 →
